## Structural Observations — Needs Manual Review

### 1. knowledge_sharing_system.py vs knowledge_transfer_system.py

Both are real implementations (~289 and ~310 lines). Both do cross-agent knowledge management with different storage backends:
- `knowledge_sharing_system.py` — `knowledge_base.json` + `agent_experiences.json`
- `knowledge_transfer_system.py` — `experience_buffer.json` + `knowledge_graph.json`

Neither has external imports. **Manual review recommended** to determine if these should be merged into a single canonical module.

### 2. adaptive_communication.py vs agent_communication_protocol.py

Different concerns:
- `adaptive_communication.py` — communication style/personality adaptation (60 lines, no imports)
- `agent_communication_protocol.py` — full inter-agent messaging protocol (342 lines, active)

Not duplicates. Both kept.

### 3. core/infrastructure/ vs /infrastructure/

`core/infrastructure/` (13 dirs) is a subset of `/infrastructure/` (31 items). `/infrastructure/` is the canonical location with CI/CD, k8s, terraform, and active bot scripts. The `core/infrastructure/` subtree contains the worker agents and bridge/teams code that appears unique (not duplicated at the top level). No duplication action taken.

### 4. core/apps/claude-code/

Contains a snapshot of Claude Code skills (`everything/` subdirectory). These are not imported by any Python code and appear to be archived skill templates, not the active `~/.claude/skills/` installation. Not deleted (no Python code to audit, contains YAML/markdown/scripts). Consider removing if disk space is a concern.

### 5. core/research/ — zero external imports

All 16 remaining research files have zero external imports into them. They may be standalone scripts only. Consider adding them to `__init__.py` exports or documenting that they are run as scripts, not imported as modules.

## Diagnostic Pattern

The swarm's automode task runner (HERMES, Pi.dev, Researcher agents) generated hundreds of "Proof of Work" files as task completion evidence. These files contained either:
1. A JSON metadata header only (10-line format in core/research/)
2. A `# TODO: Implement actual functionality` Python stub (14-line format in core/)

Neither type contained real implementation. The automode system should be configured to require actual implementation before marking tasks as completed POW deliverables.
